# design/50-ui — model, update, render

> Spec: [spec/40](../spec/40-rendering.md), [spec/50](../spec/50-keybindings.md)

## Model

```go
package ui

type Model struct {
    InitCol  int      // captured at startup
    Width    int      // terminal columns
    Height   int      // terminal rows

    Choices  []string // immutable, post-loader output
    Visible  []string // sliding window over Choices (rotate-in-place)
    Idx      int      // selection within Visible
    Limit    int      // dynamic; capped at MaxLimit
    Input    string   // current typed input
    Cursor   int      // caret offset within Input
    Pasting  bool     // inside bracketed paste

    MaxLimit int      // default 15
}
```

## Update

```go
func (m *Model) Update(ev keys.Event) Cmd
```

Exhaustively handles:

- `RuneEvent`, `KeyEvent{Backspace}`, `KeyEvent{CtrlU}`: mutate Input,
  recompute Visible := AndFilter(Choices, Tokenize(Input)), reset Idx
  to 0.
- `KeyEvent{Up}`: scroll up + decrement Idx (with rotate-in-place).
- `KeyEvent{Down}`: scroll down with multi-line eviction.
- `KeyEvent{PageUp/Down}`: rotate by Limit.
- `KeyEvent{Home}`: Visible := AndFilter(Choices, …); Idx = 0.
- `KeyEvent{End}`: rotate so the last match is bottom; Idx = Limit-1.
- `KeyEvent{Enter}`: emit `SubmitCmd(focused or Input)`.
- `KeyEvent{Esc}`, `KeyEvent{CtrlC}`: emit `CancelCmd(Input)`.
- `PasteEvent`: append Payload to Input.
- `ResizeEvent`: update Width/Height; force re-render.

The update function is **pure** — given (Model, Event) it returns the
new Model and an optional Cmd. This is the bubbletea contract and what
makes the model unit-testable without spinning a TTY.

## Render

```go
type Frame struct {
    Pre        string  // bytes to emit before drawing (clears prior frame)
    Body       string  // the new visible list
    Post       string  // cursor restoration
}

func (m *Model) Render() Frame
```

The body is constructed line-by-line:

```
input row:  ansi.cursor.to(initCol, currentRow) + erase.lineEnd + input
choice 0:   "\n" + pointer(idx == 0) + visible[0]   (multi-line allowed)
choice 1:   "\n" + pointer(idx == 1) + visible[1]
...
```

`Pre` is computed by tracking how many rows the *previous* frame
occupied, and emitting:

```
ansi.cursor.down(prevSize) + (erase.line + cursor.prevLine) × prevSize
+ ansi.cursor.to(initCol, currentRow) + erase.lineEnd
```

`Post`:

```
ansi.cursor.up(currentSize) + ansi.cursor.to(initCol + Cursor, currentRow)
```

This is exactly the legacy `clear()` and `restore()` shape, but pulled
out as a pure function over (prevSize, currentSize) so it is unit-
testable.

## Throttle

```go
type Throttle struct{ … }
func (t *Throttle) Fire(now time.Time, do func()) bool
```

Leading-edge throttle with a 72 ms window. The bubbletea program
intercepts every `tea.Msg` and only writes a frame to TTY if
`throttle.Fire(time.Now(), …)` is true. Pending events still update
the model — the throttle only affects the *output*, never the
state machine. So a paste of 100 chars updates state 100 times but
draws ~5 frames.

## Golden-frame tests

`internal/ui/render_test.go` builds a Model, asks for a Frame, and
compares the byte string against a checked-in golden. Updating goldens
is done via `go test -update` (a flag on the test binary, not an env
var, so accidents are unlikely).

## Bubbletea boundary

The bubbletea `tea.Program` is wrapped by our custom output writer
(`tty.Writer`) so frames go to `/dev/tty` instead of stdout. We never
use `tea.WithAltScreen()` because we are inline. We use
`tea.WithoutCatchPanics()` because we want our own `recover()` in main
that restores termios before re-panicking.
